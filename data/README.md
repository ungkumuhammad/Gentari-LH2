# `data/` — LH2 Techno-Economic Database (KR1.1 v1)

> Data dictionary and index for the LH2 value-chain component database.
> Every figure here is **cited** to a `references.csv` id or **tagged** per
> [`CLAUDE.md`](../CLAUDE.md) §4 and [`docs/conventions.md`](../docs/conventions.md).
> Built from in-repo Kawasaki/IAE sources only (proprietary tier). LH2 first;
> NH3 columns are added later for KR1.3.

## Tables

| File | Segment | What it holds | Primary source(s) |
|------|---------|---------------|-------------------|
| `properties/liquefaction.csv` | Liquefaction | SEC, power, train capacity/count, min scale, cycle, O–P catalyst, BOG, losses | `kawasaki-2026-questionnaire`, `-supplemental` |
| `properties/terminals.csv` | Loading/export terminal | BOR, tank size/count, dead-volume, loading-arm TRL | `kawasaki-2026-supplemental`, `-questionnaire` |
| `vessels/lh2-carriers.csv` | Shipping | Carrier capacity/speed/fleet, BOG-as-fuel, no reliquefaction, load/unload, yard | `kawasaki-2026-supplemental`, `-questionnaire`, `kawasaki-hydrogen-activities` |
| `properties/regas.csv` | Regasification | BOR, tank size/count, vaporiser type, heat duty | `kawasaki-2026-questionnaire`, `-supplemental` |
| `properties/lh2-properties.csv` | Physical properties | NBP, density, LHV/HHV, gas density | `docs/conventions.md` (pending NIST/CODATA) |
| `costs/lh2-cost-stack.csv` | Cost | IAE per-component supply-cost stack (LH2 Base/Large/Tech) + disclosed gaps | `kawasaki-2026-supplemental` → `iae-2019-gigaton` |

`lh2-database.xlsx` is a one-workbook mirror of all CSVs (one sheet each, plus a
`references` sheet), regenerated by [`../scripts/build_db_workbook.py`](../scripts/build_db_workbook.py).
The CSVs are the source of truth; the workbook is for sharing.

## Schema (tidy / long format, shared by every table)

| Column | Meaning |
|--------|---------|
| `segment` | Value-chain segment (`liquefaction`, `loading_terminal`, `shipping`, `regas`, `properties`, `cost`) |
| `component` | Equipment / chain element the row describes |
| `parameter` | The quantity or attribute |
| `value` | Point value (blank if only a range, or if the figure is an undisclosed gap) |
| `unit` | SI/metric unit per `docs/conventions.md` (blank for qualitative values) |
| `low`, `high` | Range bounds when a source gives a range instead of a point |
| `basis` | Conditions / system boundary / scenario (e.g. `Base 2.5B Nm3/y`, `at NBP`) |
| `source_id` | A `references.csv` id; blank only when `tag` is non-`cited` |
| `tag` | One of `cited`, `ASSUMPTION`, `ESTIMATE`, `needs-source` |
| `notes` | Provenance detail (e.g. KHI reply number) and caveats |

### Invariant (enforced by `tests/test_data_tables.py`)

Every row has **either** a `source_id` that resolves to `references.csv`
**or** a `tag` in {`ASSUMPTION`, `ESTIMATE`, `needs-source`}. No untagged,
unsourced numbers — the cardinal rule of `CLAUDE.md` §4.

## Tag vocabulary

- **`cited`** — backed by a `references.csv` id in `source_id`.
- **`ASSUMPTION`** — a value deliberately chosen to proceed; rationale in `notes`.
- **`ESTIMATE`** — a rough figure that must be replaced with a sourced one.
- **`needs-source`** — a figure a stakeholder declined to disclose, or a standard
  value awaiting a formal citation. Carried as an explicit gap, never invented.

## Known caveats

- **Cost stack units.** The IAE per-component figures (`costs/lh2-cost-stack.csv`)
  are reproduced from a KHI slide where the axis is **unlabelled**. Magnitudes
  (LH2 Base total ≈ 37.6) match the METI/IAE ¥/Nm³ supply-cost framing, so the
  unit is stored as `JPY/Nm3` (~2019 vintage) and every row carries a
  `[needs source: primary]` note. **Verify the unit and cost-year before using
  these in USD/kg LCOH.** No conversion to USD/kg is applied here.
- **Undisclosed CapEx/OpEx.** KHI stated a Feasibility Study is required for
  liquefaction/regas CapEx & OpEx in USD/kg and for the carrier newbuild cost.
  These are present as `needs-source` rows so the gap is visible, not silent.
- **Physical properties.** `lh2-properties.csv` values come from
  `docs/conventions.md` and are tagged `needs-source` pending the user's approval
  to add a standard external reference (NIST/CODATA/ISO).
